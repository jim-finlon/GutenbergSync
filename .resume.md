# Resume Guide

## Project Context

**Project:** GutenbergSync  
**Purpose:** .NET application to mirror Project Gutenberg archive and create RAG-ready chunks  
**Status:** Planning & Documentation Complete, Ready for Implementation

## Quick Reference

### Documentation
- **Requirements:** `docs/REQUIREMENTS.md` (v1.2)
- **Technical Spec:** `docs/TECHNICAL_SPEC.md` (v1.2)
- **README:** `README.md`

### Key Features
1. Metadata-first sync strategy
2. Incremental extraction
3. Selective extraction (by author, subject, date, book IDs)
4. Multiple export formats (JSON, Parquet, Arrow)
5. Quality validation and metrics
6. Health/status monitoring
7. Database maintenance
8. Configuration validation

### Technology Stack
- .NET 8.0 LTS
- C# 12
- SQLite (catalog)
- rsync (sync)
- Serilog (logging)
- Spectre.Console (CLI)
- System.CommandLine (CLI parsing)

## Key Design Patterns

- **Interface-based design**: All services have interfaces for testability
- **Dependency injection**: Use Microsoft.Extensions.DependencyInjection
- **Async/await**: All I/O operations are async
- **Progress reporting**: Use IProgress<T> for long operations
- **Configuration-driven**: All behaviors configurable via JSON/YAML

## Important Notes

- rsync must be auto-detected (Windows: WSL/Cygwin)
- Database defaults to `{targetDirectory}/gutenberg.db`
- Incremental extraction is default behavior
- Configuration validated on startup
- All operations should be idempotent
